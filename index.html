Calendly (widget — no JWT)
Uses the same embed token as chat (paste above). Links Calendly to the embed owner account. Run ngrok http 5000, paste the real https host + /api/webhooks/calendly (not YOUR-SUBDOMAIN), then Register webhook (after Connect). Or set CALENDLY_WEBHOOK_TARGET_URL in .env and reconnect OAuth. Bookings only update chat when Calendly POSTs to that URL (intent buy/sell does not matter). Use the Open tracked scheduling link from the chat widget (it adds utm_content); opening a plain Calendly URL may not match the conversation unless the invitee email matches your lead. If ngrok shows no POST after a real booking, the Calendly user who connected OAuth must be the same as the host of calendly_link, or use org-scope webhooks / CALENDLY_WEBHOOK_REGISTER_SCOPE=user as needed. If POSTs return 401, fix CALENDLY_WEBHOOK_SIGNING_KEY or set CALENDLY_SKIP_WEBHOOK_SIGNATURE=true in dev only. List webhooks shows what Calendly has stored; Simulate booking tests your DB without Calendly (non-production only).
